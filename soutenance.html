Parcours BUT Informatique – Spécialisation Gestion et Exploitation des Données
Objectif : Démontrer l’acquisition des compétences AC34, AC35, AC36 à travers des projets concrets

Gérer

Concevoir, gérer, administrer et exploiter les données de l’entreprise et mettre à disposition toutes les informations pour un bon pilotage de l’entreprise

AC34.01 – Capturer et stocker des ensembles volumineux et complexes de données hétérogènes

  • SAE Pharmaguide/MedicSearch (S5-S6) – Projet principal
    Lors de la SAE Pharmaguide/MedicSearch (S5-S6), j'ai conçu et géré une architecture multi-bases pour capturer des données médicamenteuses complexes : MongoDB pour les documents JSON semi-structurés, PostgreSQL pour les données relationnelles normalisées, et Neo4j pour les relations graphiques, traitant des milliers de médicaments avec leurs propriétés hétérogènes.
  • R5 C 05 - BD NoSQL
    Lors des TP NoSQL (R5 C 05), j'ai manipulé des ensembles de données volumineux et hétérogènes : 40,000 routes aériennes avec Neo4J, données documentaires JSON avec MongoDB, sessions utilisateurs avec Redis, et datasets restaurants avec Cassandra.
  • R6 C 05 - Administration des bases de données
    Lors des TP d'administration BD (R6 C 05), j'ai analysé et géré des ensembles de données complexes : rétroconception complète du schéma HR avec 7 tables relationnelles, création d'une table de test volumineuse (1.2M lignes), et maîtrise de l'architecture Oracle avec gestion des tablespaces et allocation de blocs.
  • R5 C 04 - Web (Analyse de données avec décisions)
    Lors des projets d'analyse avec Tableau (R5 C 04), j'ai intégré des ensembles de données hétérogènes : démographiques, géographiques, médicales et environnementales, avec création de relations complexes entre multiples sources CSV.
  • R5 C 07 - Données massives
    Lors des TP de données massives (R5 C 07), j'ai manipulé des ensembles de données volumineux et hétérogènes : datasets génomiques avec 1 million de marqueurs, caractéristiques botaniques multi-dimensionnelles (40 espèces, 16 caractéristiques), et propriétés chimiques de vins avec différents types de variables (continues, discrètes).
  • R5 C 08 - Techniques d'intelligence artificielle
    Lors des TP d'intelligence artificielle (R5 C 08), j'ai manipulé des ensembles de données complexes : dataset Iris avec 150 échantillons botaniques multi-dimensionnels, et étudié des cas théoriques de données génomiques volumineuses (1M de marqueurs).
  • R5 C 11 - Optimisation des données et des systèmes
    Lors du module d'optimisation des systèmes décisionnels (R5 C 11), j'ai conçu des architectures de données complexes : modélisation dimensionnelle pour données de pollution, intégration de sources géographiques et temporelles multiples, avec optimisation des performances d'interrogation.
  • R5 Admin 06 - Exploitation de la base de données
    Lors des TD d'exploitation de bases de données (R5 Admin 06), j'ai modélisé des schémas multidimensionnels complexes avec conception d'architectures en étoile adaptées aux besoins décisionnels.

AC34.02 – Préparer et extraire les données pour l'exploitation

  • SAE Pharmaguide/MedicSearch (S5-S6) – Projet principal
    J'ai développé un pipeline ETL complet : extraction de données web par scraping, transformation via scripts Python (pandas, psycopg2), chargement dans 3 SGBD différents, et association de données via fichiers Excel pour créer des relations complexes médicaments-effets indésirables.
  • R6 C 05 - Administration des bases de données
    J'ai développé des systèmes complets d'extraction automatisée : procédures PL/SQL pour rétroconception de schéma, extraction de métadonnées Oracle via vues système, scripts de volumétrie dynamique vs statique, et requêtes SQL complexes sur le dictionnaire de données.
  • R5 C 05 - BD NoSQL
    J'ai développé des systèmes d'extraction et de préparation : fonctions Python pour Redis avec validation des entrées, requêtes MongoDB pour l'extraction documentaire, et import optimisé CSV vers Cassandra.
  • R5 C 04 - Web (Analyse de données avec décisions)
    J'ai préparé et transformé les données brutes : calculs d'effectifs réels à partir de proportions, filtrage temporel et géographique, gestion des valeurs manquantes.
  • R5 Admin 06 - Exploitation de la base de données
    J'ai maîtrisé l'extraction de données décisionnelles : requêtes MDX pour l'analyse multidimensionnelle, extraction de mesures par dimensions, et filtrage de données selon critères métier.

AC34.03 – Appliquer des méthodes d'exploration et d'exploitation des données

  • R5 C 08 - Techniques d'intelligence artificielle – Projet principal
    J'ai appliqué un large éventail de méthodes d'exploration : 5 algorithmes supervisés (k-NN, Naïve Bayes, LDA, QDA, SVM) avec validation croisée 10-fold, 3 algorithmes non supervisés (k-means, CHA, GMM), analyse comparative des performances avec métriques multiples, et évaluation détaillée via matrices de confusion.
  • SAE Pharmaguide/MedicSearch (S5-S6)
    J'ai appliqué des méthodes avancées d'exploration : requêtes Cypher pour l'analyse de graphes médicamenteux, recherche approximative avec algorithmes de similarité, visualisations interactives Neo4j, et interface web avec filtres dynamiques.
  • R5 C 07 - Données massives
    J'ai appliqué des méthodes d'exploration de données massives : ACP pour réduire 1 million de variables génomiques à 2 composantes principales, clustering sur datasets botaniques et vinicoles, calculs de variance expliquée, et visualisations interactives 2D/3D.
  • R5 C 11 - Optimisation des données et des systèmes
    J'ai appliqué des méthodes d'exploration avancées : analyses géospatiales des concentrations de pesticides, KPI automatisés avec filtres interactifs, tableaux de bord multi-niveaux (national/régional), et visualisations dynamiques avec interactions configurées.
  • R5 C 04 - Web (Analyse de données avec décisions)
    J'ai appliqué des méthodes d'exploration avancées : analyses géospatiales, corrélations statistiques, comparaisons multidimensionnelles, avec visualisations interactives.
  • R5 C 05 - BD NoSQL
    J'ai appliqué des méthodes d'exploration avancées : requêtes Cypher pour l'analyse de graphes (top compagnies aériennes, répartition géographique), gestion de sessions temporelles avec Redis, et analyses relationnelles avec Cassandra.
  • R5 Admin 06 - Exploitation de la base de données
    J'ai appliqué des méthodes d'exploration décisionnelles : opérations drill-down/drill-up, analyses croisées multi-dimensionnelles, analyses temporelles saisonnières, et génération de graphiques automatisés.

AC34.04 – Mettre en production et optimiser le système de gestion de données

  • SAE Pharmaguide/MedicSearch (S5-S6) – Projet principal
    J'ai mis en production un système complet d'information médicamenteuse : déploiement Docker multi-conteneurs, application Flask avec authentification et autorisation, interface d'administration, système de recherche optimisé avec cache, et architecture scalable préparée pour le cloud (Azure/AWS).
  • R6 C 05 - Administration des bases de données
    J'ai optimisé des systèmes Oracle en production : création d'index composés performants (35 Mo pour 1.2M lignes), configuration optimale des paramètres de stockage (pctfree/pctused), calculs de performance disque avec latence et débit, et choix d'architecture mémoire selon contraintes système.
  • R5 C 05 - BD NoSQL
    J'ai mis en production des systèmes NoSQL complets : cluster Cassandra multi-nœuds avec Docker, Redis configuré en réseau avec persistance, MongoDB optimisé avec index, et Neo4J avec résolution de conflits de ports.
  • R5 C 04 - Web (Analyse de données avec décisions)
    J'ai mis en production des tableaux de bord Tableau fonctionnels et optimisés, avec interfaces utilisateur interactives et navigation intuitive.
  • R5 Admin 06 - Exploitation de la base de données
    J'ai mis en production un système décisionnel complet Mondrian/Tomcat, avec interfaces JPivot fonctionnelles et serveur opérationnel.

AC35.01 – Mesurer les impacts économiques, sociétaux et technologiques

  • SAE Pharmaguide/MedicSearch (S5-S6) – Projet principal
    J'ai analysé les impacts du projet MedicSearch : coûts d'infrastructure cloud estimés, bénéfices sociétaux (accès démocratisé à l'information médicamenteuse), choix technologiques justifiés par les performances (Neo4j pour les relations vs PostgreSQL pour les requêtes), et optimisations économiques (cache Mistral AI, architecture managée).
